# UX для системы посещаемости

Смотрю на ваш интерфейс — это система управления тренировками/расписанием. Вот мой анализ:

## Лучший подход: **Не отдельная страница, а встроенная в расписание**

### Почему не стоит делать отдельную страницу для отметки

Тренер открывает расписание → видит тренировку → хочет отметить посещаемость прямо здесь. Лишний переход в "Посещение" разрывает этот поток.

---

## Рекомендуемая архитектура

### 1. **Расписание** — точка входа для отметки
```
Карточка тренировки → кнопка "Отметить посещаемость"
→ открывается модальное окно / боковая панель (не новая страница)
со списком спортсменов группы + чекбоксы
```

### 2. **Посещение** (уже есть в меню) — только для **просмотра и аналитики**
- Статистика по группам / спортсменам
- Фильтрация по датам
- Экспорт
- Пропуски и причины

---

## Конкретный UX паттерн для карточки тренировки

**Состояния карточки:**
- 🔵 Запланирован — тренировка ещё не началась
- 🟡 Идёт сейчас — кнопка **"Отметить"** становится активной
- ✅ Отмечено — показывает `12/15` (посетили / всего)
- ⚠️ Не отмечено — если тренировка прошла, но посещаемость не заполнена

---

## Структура страницы "Посещение" (просмотр)

```
Посещение
├── Фильтры: Группа / Период / Спортсмен
├── Сводная таблица (группа × дата)
│   ├── Зелёный = был
│   ├── Красный = не был
│   └── Серый = тренировки не было
└── Детальный вид по спортсмену
    └── % посещаемости за месяц
```

## Best practices из спортивных систем

| Практика | Зачем |
|---|---|
| Быстрая отметка "все присутствуют" + отмечаем отсутствующих | 95% случаев все пришли — не заставляй кликать каждого |
| Причина пропуска (болезнь / без причины / соревнование) | Важно для аналитики |
| Отметка прямо с телефона | Тренер на площадке, не за компьютером |
| Возможность исправить посещаемость задним числом | Тренер забыл отметить — нужен доступ к прошлым тренировкам |
| Push-уведомление "Не забудьте отметить" после окончания | Снижает процент незаполненных тренировок |

## Итог

- **Отметка** → модалка/слайдер прямо из расписания
- **Просмотр/аналитика** → отдельная страница "Посещение"
- Отдельная страница для **отметки** — лишняя, только усложнит навигацию
